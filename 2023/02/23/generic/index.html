泛型

自从以前学习泛型之后就一直没有使用过，或者说在项目的开发中没有写过泛型的模型。随着学习的深入也发现了自己的不足，故写下这篇文章作为总结。

1. 什么是泛型

泛型：即参数化类型

顾名思义，就是将类型由原来的具体的类型参数化，类似于方法中的变量参数，此时的类型定义成参数形式，（可以称为类型形参），然后在使用/调用时传入具体的类型（类型实参）。

泛型的本质是为了参数化类型，（在不创建新的类型的情况下，通过泛型指定不同类型来控制形参具体限制的类型）。也就是说在泛型使用过程中，操作的数据类型被指定为一个参数，这种参数可以在用在类、接口和方法中，分别称为：泛型类、泛型接口、泛型方法。

例如：GenericClass{}

常用的泛型类型变量：

E：元素（Element），多用于框架

K：关键字（Key）

N：数字（Number）

T：类型（Type)

V：值（Value）

?：通配符

在上面代码中可以看出，在我们将一个对象放入集合中，集合不会记住对象的类型，当从集合中取对象时，该对象的编译类型变成了Object类型。

但如果都是Object类型我们使用的话就需要向下转型。如果向下转型在代码运行期间就很容易出现异常ClassCastException（类型转换异常）所以使用泛型来解决这个问题。

所以泛型的意义在于：

  1. 适用于多种数据类型执行相同的代码（代码复用）
  2. 泛型中的类型在使用时指定，不需要强制类型转换（类型安全，编译器会检查类型）

4. 泛型接口

定义一个泛型接口：public class GenericInterface<T>{}

实现泛型接口方式一：public class ImplGenericInterface1<T> implements GenericIntercace<T>

实现泛型接口方式二：public class ImplGenericInterface1<T> implements GenericIntercace<T>

从上面两种实现方式可以看出：

  1. 实现类ImplGenericInterface1可以不指定实现的参数类型，而使用泛型直接实现接口的方法。但是实现类本身也必须是泛型类，由于是泛型类 在new的时候必须指定其类型.
  2. 实现类ImplGenericInterface2可以在实现接口的时候定义接口的实现类型，这样实现类就可以不定义为泛型类而实现接口的方法，在new的时候也不用指定参数类型。

6. 泛型的类型限制

如果在方法体中需要运算，使用泛型会导致简单的运算都不能执行。因为泛型的类型是在方法使用的时候指定的

计算最小值, compareTO()方法在编译时期就会报错，如果要实现这样的功能就需要对泛型方法的类型做出限定。

限定类型使用extends关键字指定，可以指定类、接口，类放在前面，接口放在后面用&符号分割。

例如：<T extends ArrayList & Comparable<T> & Serializable>

再看一个例子:

  1. 首先ArrayList是List的子类，但是ArrayList也是个泛型类。虽然创建了ArrayList<String>和ArrayList<Integer>但是都能添加到TypeLimitForClass<ArrayList>中。说明在类上做泛型的限制，如果传递的参数也是泛型类则不会限制泛型类的泛型类型。

  2. 在方法getMinListSize(T a, T b)中限制了传递参数必须继承Comparable类型。所有数字相关的类都实现Comparable接口。
